# Triple-Platform System

AiAgentArchitect Lite ships entities for three host platforms simultaneously: Google Antigravity, Claude Code, and OpenAI Codex.

## Source of truth: `.agents/`

Every entity lives in `.agents/` first. This is the canonical location. Antigravity reads `.agents/` directly.

```
.agents/
├── workflows/        ← wor-*.md, age-*.md
├── skills/           ← ski-*/SKILL.md
├── rules/            ← rul-*.md
├── knowledge-base/   ← kno-*.md
├── resources/        ← res-*.md
├── hooks/            ← hok-*.md (root-level only)
├── scripts/          ← scp-*.md or scripts
└── layers/           ← {layer-id}/MANIFEST.yaml + sub-tree
```

## Mirror: `.claude/`

Claude Code requires entities at specific paths. `bash scripts/sync-dual.sh --agents-to-claude --prune` mirrors `.agents/` to `.claude/` with the path remapping it expects:

| `.agents/` path | `.claude/` path |
|---|---|
| `workflows/wor-*.md` | `commands/wor-*.md` |
| `workflows/age-*.md` | `agents/age-*.md` |
| `workflows/com-*.md` | `commands/com-*.md` |
| everything else | identical relative path |

The sync is bidirectional but only at the root level (layer entities should always be edited in `.agents/layers/{X}/`).

## Compiled output: `.codex/`

`python3 scripts/build-codex.py` compiles `.agents/` into Codex's expected format:

- Behavioral entities (`wor-*`, `age-*`, `com-*`) → TOML agent definitions in `.codex/agents/`.
- Procedural entities (`ski-*`, `rul-*`, `kno-*`, `res-*`) → direct copy.
- Hooks → `.json` fragments merged into `.codex/hooks.json`.

Codex doesn't support live hooks, so any layer that depends on hooks (none in Lite, since the bundled layers are orchestrator-driven) degrades to manual invocation in Codex.

## Context roots

The `CLAUDE.md` and `AGENTS.md` at repo root are auto-generated by `python3 scripts/build-context-roots.py` from `templates/CLAUDE.md.tpl` and `templates/AGENTS.md.tpl` plus the active layer sections. Re-run after editing layer manifests.

## When to re-sync

After editing entities in `.agents/`:

```bash
bash scripts/sync-dual.sh --agents-to-claude --prune
python3 scripts/build-codex.py
python3 scripts/build-context-roots.py
```

The sync scripts are idempotent — safe to re-run.

## Selecting platforms during install

The install wizard detects which platforms you have available (`~/.claude/`, `~/.codex/`) and pre-selects them. You can opt out of any platform; the corresponding mirror/compile step is then skipped.

If you only enabled Claude Code at install but later want Codex, re-run `bash install.sh` and tick `codex`.
