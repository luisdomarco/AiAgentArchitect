## User Story

**As** a sales agent
**I want** to build complete, real-time financing simulations with configurable quote items, customer financial profile, and a comparative history of simulations,
**So that** I can present tailored financing proposals per customer profile, systematically compare alternatives, and share selected results with the customer

## Definition

**Brand:** Global
**Jobs:** Sales agent
**Modules:** Financing simulator

**Related:** 

-

**Problem/Need**

Current financing simulator doesn't cover
- Make simulations in real time
- Consider simulations without and with financing insurance
- Configure quote items between all catalog of Services, Accessories, Discounts to make a more certain simulation
- Configure customer financing profile with basic data to make a more certain simulation
- Current history of generated simulations don't facilitate full comparation following agent needs
  - Simulation A with Simulation N
  - Same interest rate, different quote
  - Different interest rate, same quote
  - Different interest rate, different quote
  - Same simulation, different monthly payments of the terms
  - Same simulation, with/without insurance monthly payments
- Send simulations to customer: 
  - One full simulation
  - N full simulations
  - Within these options
    - Specific terms from N simulations, it can be different between them
    - Terms-Monthly payments Only with / Only without insurance

**Scope**

- AS IS - Maintain current Financing modes, formulas and main fields: Interest rate, Down payment, Total operation amount, Financing amount, Estimated km (Lease)
- Implement Quote Items configuration within a simulation to add, remove, mark as free, enable or disable Services, Accessories and Discounts from the quote catalog (insurance excluded).
  - Actions on these: Add, Remove, Mark as free, Enable/Disable
  - Management access depends on user permissions:
    - Visibility of the section and ability to manage items require at least one of these permissions: Manage Services, Manage Accessories, Manage Discounts.
    - If no permission is present, the section is hidden.
- Implement customer fields to configure the basic data of the customer financing profile:
  - Employment status: 
    - self_employed, permanent_contract, temporary_contract, permanent_seasonal_contract, civil_servant, retirement_pension, disability_pension, survivors_pension, other_pension, investment_income, student, unemployed.
    - Visibility and Editability behavior:
      - Requires "View" permission for Labour Situation to see the section.
      - Requires "Edit" permission to modify the fields. 
      - If "View" is granted but "Edit" is missing, fields are visible but disabled.
      - If "View" is missing, the section is hidden regardless of "Edit" status.
  - Net monthly salary: Owner
  - Number of salary payments per year: Owner
  - Net monthly salary: Co-owner (optional)
  - Number of salary payments per year: Co-owner (optional)
  - Max monthly payment: auto-calculated field using formula:
    `Formula = (((Net monthly salary: Owner) X (Number of salary payments per year: Owner)) + ((Net monthly salary: Co-owner) X (Number of salary payments per year: Co-owner))) X 25%`
- **Data Persistence & Pre-filling:**
  - Owner's financial data (salary, payments) is stored at the Customer profile level.
  - If data already exists in the customer profile, it is pre-filled in the simulation.
  - If data is empty or modified and then saved within the simulation, it is updated back to the main Customer profile.
  - Co-owner data and Labour Situation are stored at the financing simulation level.
- Give visual visibility to the sale agent about which terms/monthly-payments combinations the customer can or cannot afford based on the "Max monthly payment" calculated.
- Enable real-time recalculation of the simulation when any parameter is modified by the agent.
- Enable visual layout to display monthly payment variants for terms with and without financing insurance.

**History & Sharing:**
- Enhance the simulations history to support full side-by-side comparisons (Same/Different interest rates, quotes, terms, with/without insurance).
- Implement selective "Send to Customer" allowing the agent to pick specific terms and filter by insurance status.


**Out of scope**

Modifications to the existing financing modes calculation formulas or core validation rules (AS IS logic is preserved).
Modifications to the existing fields Interest rate, Down payment, Total operation amount, Financing amount, Estimated km (Lease)

## Acceptance Criteria

- **Feature:** Quote Items Configuration

- **Background:** Given the sales agent is configuring a financing simulation

- **Scenario:** Configure Quote items in the simulator, on permissions (Happy Path)
  - Given the agent has at least one of the permissions that gives the capacity to list Services, Accessories, Discounts
    - Accessories > List OR Services > List OR Discounts > List
  - When the agent opens the simulator Then
    - the section to manage these items is visible
    - the user can list the items for which he has permission
  - When the agent has permission to Create < item > Then the agent can add < item > to simulation
  - When the agent has permission to Edit < item > Then the agent can edit < item >
  - When the agent has permission to Remove < item > Then the agent can remove < item >

- **Scenario:** Denied access to Quote items management (Unhappy Path)
  - When the agent does not have any of the "List" permission for Services, Accessories, Discounts Then the user doesn't see the section
  - When the agent does not have "List" permission for one of these items Then the user cannot see these items in the section
  - When the agent has not permission to Create < item > Then the agent cannot add < item > to simulation
  - When the agent has not permission to Edit < item > Then the agent cannot edit < item >
  - When the agent has not permission to Remove < item > Then the agent cannot remove < item >

- **Scenario:** Add and enable a new quote item successfully
  - Given the agent has found an available service, accessory, or discount
  - When the agent adds it to the simulation Then agent can enable / disable it application
  - When the agent enables it Then the item applies on the real time simulation
  - When the agent disables it Then the item doesn't applies on the real time simulation
  - When the agent enables / disables it Then the simulation totals recalculate automatically to reflect the action

- **Scenario:** Mark a configured quote item as free
  - Given a service or accessory is added and enabled in the simulation
  - When the agent marks the item as "Free"
  - Then the item remains listed in the section where services, accessories, and discounts are configured inside the simulator
  - And its contribution to the simulation becomes zero
  - And the simulation totals recalculate automatically


- **Feature:** Real-time Simulation calculation

- **Scenario:** Initial real-time simulation display based on rule configuration
  - Given the parameters match an applying financing rule
  - When the agent is viewing the real-time simulation
  - Then the simulation displays all terms that match the financing rule
  - And if the rule is configured "without insurance", it displays only the "Financing monthly payment", "Service monthly payment" (if applicable), and "Total monthly payment"
  - And if the rule is configured "with insurance", it displays only the "Financing monthly payment", "Service monthly payment" (if applicable), and "Total monthly payment"
  - And if the rule is configured "without insurance" and "with insurance", it displays two variants per term:
    - "Financing monthly payment without insurance" + "Service monthly payment" + "Total monthly payment"
    - AND "Financing monthly payment with insurance" + "Service monthly payment" + "Total monthly payment"

- **Scenario:** Instant recalculation upon parameter change
  - Given the real-time simulation shows results
  - When the agent modifies any parameter (Down payment, Quote item, Rule, etc.)
  - Then all financing amounts and term monthly payments update instantaneously without page refresh


- **Feature:** Customer Labour situation

- **Scenario:** New Labour situation section is implemented in the customer and the user can manage it
  - Given a sales agent is in the customer view
  - When the agent has permission "Customer > Labour situation - View" = Yes / Owner
  - Then new section "Labour situation" is visible after "Basic data"
  - And in this section are considered these fields
    - Social security number
    - Employment status
    - Net monthly salary
    - Number of salary payments per year
  - When the agent has permission "Customer > Labour situation - Edit" = Yes / Owner
  - Then the agent can edit the fields of this section

- **Scenario:** Denied access to see Labour situation section
  - Given a sales agent the agent has permission "Customer > Labour situation - View" = No / Empty
  - When the agent is in customer view
  - Then the section "Labour situation" is not visible for them
  
- **Scenario:** Denied access to manage Labour situation section
  - Given a sales agent the agent has permission "Customer > Labour situation - Edit" = No / Empty
  - When the agent is in customer view
  - And the section "Labour situation" is visible for them
  - Then the section "Labour situation" is not editable for them


- **Feature:** Customer Financing Profile

- **Background:** Given a sales agent is creating or viewing a simulation

- **Scenario:** Access to "Customer financing profile" and edit it based on permissions (Full Access)
  - Given the user has the permission "Customer > Labour situation - View" = Yes / Owner
  - And the user has permission "Customer > Labour situation - Edit" = Yes / Owner
  - And the user has permissions to view and manage the financing simulatior
  - When the user is in the financing simulatior
  - Then "Customer financing profile" section and fields are visible and editable for them

- **Scenario:** View-only access to "Customer financing profile"
  - Given the user has the permission "Customer > Labour situation - View" = Yes / Owner
  - And the user does not have the "Edit" permission
  - When the user is in the financing simulator
  - Then the "Customer financing profile" section and fields are visible but disabled for editing

- **Scenario:** Denied access to "Customer financing profile"
  - Given the user has the permission "Customer > Labour situation - View" = No / Empty
  - When the user is in the financing simulator
  - Then the "Customer financing profile" section and fields are are not visible for them (regardless of Edit permission)

- **Scenario:** Pre-filling owner financing profile data from customer data 
  - Given the customer already has "Employment status" AND/OR "Net monthly salary" AND/OR "Number of salary payments per year" stored in the Customer
  - When the agent opens the financing simulator
  - Then the Owner's fields are pre-filled with the existing customer data
  
- **Scenario:** Saving owner data from simulation to customer
  - Given the Customer Financing Profile fields were empty OR modified in the simulator
  - When the agent saves the these section
  - Then the Owner's financing profile fields are also saved/updated in the customer > Labour situation fields


- **Feature:** Calculate the max monthly payment based on customer financing profile

- **Scenario:** Auto-calculate Max monthly payment
  - Given valid "Employment status" AND "Net monthly salary" AND "Number of salary payments per year" for Owner and Co-owner (it can be empty)
  - When the data is saved
  - Then the "Max monthly payment" field is auto-calculated using the formula
    - Formula = (((Net monthly salary: Owner) X (Number of salary payments per year: Owner)) + ((Net monthly salary: Co-owner) X (Number of salary payments per year: Co-owner))) X 25%
  - And the affordability visual indicators are updated immediately in the real-time simulation

- **Scenario:** Visual feedback for affordable and unaffordable terms
  - Given the "Max monthly payment" is already calculated
  - When the real-time simulation displays the financing terms and monthly payments
  - The terms with a monthly payment above the Max monthly payment show a visually negative/warning indicator

- **Scenario:** Incomplete financial profile behavior
  - Given required fields for Max monthly payment calculation are missing
    - "Employment status" AND "Net monthly salary" AND "Number of salary payments per year" of the Owner
  - When the agent try to save a simulation
  - Then the simulation is calculated normally but without affordability analysis
  - And the "Max monthly payment" field remains empty


- **Feature:** Enhanced Simulation History and Comparisons

- **Scenario:** Side-by-side comparison of different simulations
  - Given multiple saved simulations are present in the history panel
  - When the agent views the history
  - Then the agent can visually contrast different simulations side-by-side (interest rates, down payments, quote items, monthly payments)

- **Scenario:** With/without insurance comparison of same/different simulations
  - Given multiple saved simulations are present in the history panel, with / without insurance
  - When the agent views the history
  - Then the agent can visually contrast 
    - same simulation: with/without insurance
    - different simulations: with/without insurance


- **Feature:** Select simulation/s to send to Customer

- **Scenario:** Send specific terms from multiple simulations
  - Given multiple saved simulations are present in the history panel
  - When the agent views the history
  - Then the agent can select entire simulations (N)
  - And the agent can select explicity terms from N simulations
  - And the agent can select within the same simulation only the terms with / without insurance
  - And the agent can select 1 entire simulation AND explicity terms from N simulations
